# Repository Boundaries

## `paygod-cloud-starter`
This repository hosts:
- Cloud demo API scaffolding
- Developer onboarding docs
- Workflow templates and contributor guidance

## `paygod-kernel-mvp`
This repository should remain protocol-focused and minimal.
Do not place cloud demo/server onboarding implementation there.

## Rule of thumb
- Cloud/developer demo work goes in `paygod-cloud-starter`.
- Protocol/kernel logic goes in `paygod-kernel-mvp`.
