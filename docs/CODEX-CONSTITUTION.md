# Codex Constitution (Contributor Conventions)

## PR conventions
- Keep changes minimal and focused.
- Include motivation and verification steps.
- Prefer small files and direct instructions for onboarding.
- Use consistent endpoint naming and clear JSON payloads.

## Commit conventions
- Use clear, imperative commit messages.
- Group related changes in one commit where practical.

## Developer friendliness
- Keep local run steps copy/paste-ready.
- Avoid hidden dependencies for demo paths.
